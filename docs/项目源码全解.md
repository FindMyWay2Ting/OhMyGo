# OhMyGo 项目源码全解

目标：让你能把这个项目“讲清楚、讲细致、讲到函数级别”，并知道每一层在做什么、数据怎么流动

> 文档基于当前仓库代码结构编写，包含后端（Go/Gin）、前端（Vue3）、中间件（MySQL/Redis/RabbitMQ）以及容器化（Docker Compose）。  
> 代码引用均以本地路径链接形式给出，便于你在 IDE 里一键跳转。

## 0. 快速抓住项目亮点（简历可用）

- **全栈 AI 聊天系统**：Vue3 前端 + Gin 后端，支持登录注册、会话管理、聊天（普通/流式 SSE）。
- **会话级上下文管理**：`userName + sessionID -> AIHelper`，一个会话一个上下文容器。
- **消息异步落库**：消息写入通过 RabbitMQ 解耦，消费者异步写 MySQL，降低主链路延迟。
- **RAG 文件问答**：文件上传后进行向量化（Embedding），存到 Redis（RediSearch），查询时做向量检索并拼 Prompt。
- **可插拔模型工厂**：`modelType` 选择 OpenAI / RAG / MCP / Ollama（接口层）。
- **TTS 语音合成**：对接百度 TTS 创建任务 + 查询任务结果。

## 2. 核心数据流（端到端）

### 2.1 登录与鉴权（Token）

1. 前端登录：`POST /api/v1/user/login`
2. 后端校验账号密码，生成 JWT： [utils/myjwt/jwt.go](file:///d:/GopherAI-v2/utils/myjwt/jwt.go)
3. 前端把 token 存 localStorage： [Login.vue](file:///d:/GopherAI-v2/vue-frontend/src/views/Login.vue)
4. 后续请求自动带 Header：`Authorization: Bearer <token>`： [api.js](file:///d:/GopherAI-v2/vue-frontend/src/utils/api.js)
5. Gin 中间件解析 token 并把 `userName` 放入 `gin.Context`： [middleware/jwt/jwt.go](file:///d:/GopherAI-v2/middleware/jwt/jwt.go)

### 2.2 新会话与旧会话切换（SessionID）

- 新会话：前端点击“新聊天” → 首次发送走 `send-new-session` 或 `send-stream-new-session` → 后端创建 session（uuid）并返回 sessionId。
- 旧会话：前端点击旧会话条目 → `currentSessionId` 切换 → 若消息未缓存则请求 `/AI/chat/history` 拉历史。

### 2.3 聊天消息落库（RabbitMQ → MySQL）

- `AIHelper.AddMessage(..., Save=true)` 时会通过 `saveFunc` 把消息序列化后发到 RabbitMQ 队列 `Message`。
- RabbitMQ 消费者 `MQMessage` 异步写 MySQL `message` 表。

核心字段：

- `model AIModel`：模型接口实例
- `messages []*model.Message`：会话历史（内存）
- `saveFunc func(*model.Message) (*model.Message, error)`：消息“持久化策略”（默认：发 RabbitMQ）

核心函数：

- `NewAIHelper(model, sessionID)`：
  - 初始化 messages
  - 默认 `saveFunc`：调用 `rabbitmq.RMQMessage.Publish(...)`
- `AddMessage(content, userName, isUser, save)`：
  - 追加到 `messages`
  - `save=true` 则调用 `saveFunc` 异步落库（队列）
- `SetSaveFunc(...)`：可替换保存策略
- `GetMessages()`：返回历史副本（避免外部修改）
- `GenerateResponse(userName, ctx, userQuestion)`：
  1) 先存用户问题（Save=true）
  2) 转成 `[]*schema.Message`（role+content）
  3) 调模型 `GenerateResponse`
  4) 存 AI 回复（Save=true）
- `StreamResponse(...)`：同上，但调模型 `StreamResponse`，并通过 cb 实时推 chunk，结束返回完整文本并落库

## 9. 后端：模型实现（OpenAI / RAG / MCP / Ollama）

### 9.1 common/aihelper/model.go（AIModel 接口与 4 种实现）

文件：[common/aihelper/model.go](file:///d:/GopherAI-v2/common/aihelper/model.go)

公共接口：

- `AIModel`：
  - `GenerateResponse(ctx, messages)`：非流式
  - `StreamResponse(ctx, messages, cb)`：流式
  - `GetModelType()`：返回类型编号

#### 9.1.1 OpenAIModel（modelType="1"）

- `NewOpenAIModel`：读取环境变量 `OPENAI_API_KEY/OPENAI_MODEL_NAME/OPENAI_BASE_URL`，创建 Eino OpenAI ChatModel
- `GenerateResponse`：`llm.Generate`
- `StreamResponse`：
  - `llm.Stream` 拿到 stream
  - 循环 `Recv()`，每段 chunk：
    - 拼接到 `fullResp`
    - 调用 `cb(chunk)` 推给前端
  - EOF 结束返回完整文本

#### 9.1.2 AliRAGModel（modelType="2"）

- `NewAliRAGModel`：读取配置 `ragModelConfig`（baseURL/chatModelName），使用 OpenAI 兼容 chat model
- `GenerateResponse/StreamResponse`：
  - 先尝试 `rag.NewRAGQuery(ctx, username)` 创建检索器
  - 用用户最后一句问题做 query，`RetrieveDocuments` 拿 topK 文档
  - `BuildRAGPrompt` 拼提示词，把最后一条 user message 替换为 prompt
  - 调 LLM 生成（若无文件/检索失败则降级为不带 RAG 的生成）

#### 9.1.3 MCPModel（modelType="3"）

- `NewMCPModel`：创建 LLM + 设定 `mcpBaseURL=http://localhost:8081/mcp`
- `GenerateResponse`：
  1) 第一次提示：让 LLM 输出固定 JSON，声明是否调用工具、工具名、参数
  2) 若需要调用工具：初始化 MCP Client → CallTool → 拿 toolResult
  3) 第二次提示：把“原问题 + 工具结果”喂给 LLM 生成最终回答
- `StreamResponse`：同理，但以流式方式输出

#### 9.1.4 OllamaModel（modelType="4"）

- `NewOllamaModel(baseURL, modelName)`：创建 Eino Ollama ChatModel
- `GenerateResponse/StreamResponse`：与 OpenAIModel 类似

## 10. 后端：消息队列（RabbitMQ）

### 10.1 common/rabbitmq/rabbitmq.go（连接/通道/发布/消费）

文件：[common/rabbitmq/rabbitmq.go](file:///d:/GopherAI-v2/common/rabbitmq/rabbitmq.go)

关键点：

- `conn` 是包级全局连接：只 Dial 一次，多个队列/实例共用同一连接。
- `NewWorkRabbitMQ(queue)`：
  - exchange 为空字符串 `""`（默认交换机）
  - key = queue 名称
- `Publish`：
  - `QueueDeclare(key)`
  - `channel.Publish(exchange="", routingKey=key)`
- `Consume`：
  - `QueueDeclare(key)`
  - `channel.Consume(...)` 拉取消息

结论：当前使用 **默认交换机（Direct）+ 单队列 `Message`** 的 Work Queue 风格。

### 10.2 common/rabbitmq/init.go（初始化全局队列与消费者）
