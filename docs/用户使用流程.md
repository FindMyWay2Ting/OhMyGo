# OhMyGo 用户使用流程（自己借助AI一起开发的小项目）

本文以“一个真实用户从打开页面到聊天、切换会话、上传文件做 RAG、图片识别、TTS”为主线，串起来说明前后端的完整调用链路与关键数据结构。

## 1. 系统组成与关键概念

### 1.1 组件

- 前端：Vue3（目录 [vue-frontend](file:///d:/GopherAI-v2/vue-frontend)）
- 后端：Go + Gin（入口 [main.go](file:///d:/GopherAI-v2/main.go)）
- 存储/中间件：
  - MySQL：用户/会话/消息表（Gorm AutoMigrate）
  - Redis：验证码、RAG 索引（使用 RediSearch FT.* 命令）
  - RabbitMQ：异步落库消息（一个队列 `Message`）
- 大模型调用：Eino 的 OpenAI 兼容接口（支持 OpenAI/DashScope 兼容）

### 1.2 两个 ID：JWT Token vs SessionID

- JWT Token：登录/注册成功后颁发，用于鉴权；前端存 localStorage，每次请求放到 `Authorization: Bearer <token>`。
- SessionID：一次“聊天会话”的唯一标识；由后端创建并返回给前端，后续聊天请求带上它来维持上下文与归档。

### 1.3 一个会话一个 AIHelper

后端内存中维护 `userName -> sessionID -> *AIHelper` 的映射（见 [AIHelperManager](file:///d:/GopherAI-v2/common/aihelper/manager.go#L10-L14)）。

## 2. 启动后端时做了什么

后端启动入口 [main.go](file:///d:/GopherAI-v2/main.go#L52-L73)：

1. 读取配置（`config/config.toml`）
2. 初始化 MySQL（并 AutoMigrate 表结构）
3. 从 DB 读取历史消息回放到内存（初始化 AIHelperManager）：[readDataFromDB](file:///d:/GopherAI-v2/main.go#L22-L50)
4. 初始化 Redis 客户端
5. 初始化 RabbitMQ，并启动消费者 goroutine
6. 启动 HTTP 服务（默认端口 9090）

注意：RabbitMQ 消费者会把消息写入数据库（见 [MQMessage](file:///d:/GopherAI-v2/common/rabbitmq/meesage.go#L29-L44)）。

## 3. 路由与鉴权规则

路由总入口：[router.go](file:///d:/GopherAI-v2/router/router.go#L9-L45)

- 统一前缀：`/api/v1`
- 不需要登录（无 JWT 中间件）：
  - `/api/v1/user/register`
  - `/api/v1/user/login`
  - `/api/v1/user/captcha`
- 需要登录（挂载 `jwt.Auth()`）：
  - `/api/v1/AI/*`
  - `/api/v1/file/*`
  - `/api/v1/image/*`

JWT 中间件： [middleware/jwt.go](file:///d:/GopherAI-v2/middleware/jwt/jwt.go#L15-L44)

- 取 token 的方式：
  - Header：`Authorization: Bearer <token>`
  - 或 URL 参数：`?token=<token>`（兼容）
- 解析 token： [ParseToken](file:///d:/GopherAI-v2/utils/myjwt/jwt.go#L33-L43)
- 解析成功后把 `userName` 写入 `gin.Context`，后续 controller 直接 `c.GetString("userName")` 获取。

## 4. 用户首次使用：注册 / 登录 / Token 存储

### 4.1 发送验证码

前端会调用：

- `POST /api/v1/user/captcha`
- body：`{ "email": "xxx@example.com" }`

后端链路：

1. controller 解析参数：[HandleCaptcha](file:///d:/GopherAI-v2/controller/user/user.go#L86-L110)
2. service 生成验证码并写入 Redis（2 分钟过期）：[SetCaptchaForEmail](file:///d:/GopherAI-v2/common/redis/redis.go#L35-L39)
3. 如果配置了邮箱 SMTP，则发送邮件；否则（本地调试模式）直接把验证码回到响应里

说明：是否“配置了邮箱”，由 [isEmailConfigured](file:///d:/GopherAI-v2/service/user/user.go#L99-L116) 决定。

### 4.2 注册

前端调用：

- `POST /api/v1/user/register`
- body：`{ "email": "...", "password": "...", "captcha": "123456" }`

后端链路：

1. controller：[Register](file:///d:/GopherAI-v2/controller/user/user.go#L66-L85)
2. service：[Register](file:///d:/GopherAI-v2/service/user/user.go)（流程：邮箱查重 → 验证验证码 → 生成 username → 写 DB → 签发 token）
3. 响应返回：
  - `token`：JWT
  - `username`：系统生成的用户名（便于后续使用用户名登录）

### 4.3 登录

前端调用：

- `POST /api/v1/user/login`
- body：`{ "username": "...", "password": "..." }`

后端链路：

1. controller：[Login](file:///d:/GopherAI-v2/controller/user/user.go#L45-L64)
2. service：[Login](file:///d:/GopherAI-v2/service/user/user.go#L13-L32)：
  - 查用户
  - `MD5(password)` 比对
  - 签发 JWT（HS256）：[GenerateToken](file:///d:/GopherAI-v2/utils/myjwt/jwt.go#L16-L31)

前端把 token 存到浏览器 localStorage： [Login.vue](file:///d:/GopherAI-v2/vue-frontend/src/views/Login.vue#L84-L92)

后续所有需要鉴权的请求都会自动带上 `Authorization: Bearer <token>`（axios 拦截器）：[api.js](file:///d:/GopherAI-v2/vue-frontend/src/utils/api.js#L8-L16)

## 5. 聊天主流程（会话列表 / 新会话 / 旧会话 / 历史）

### 5.1 进入聊天页：加载会话列表

前端进入聊天页后，会调用：

- `GET /api/v1/AI/chat/sessions`

前端逻辑： [loadSessions](file:///d:/GopherAI-v2/vue-frontend/src/views/AIChat.vue#L181-L199)

后端 controller： [GetUserSessionsByUserName](file:///d:/GopherAI-v2/controller/session/session.go#L50-L63)

后端 service： [GetUserSessionsByUserName](file:///d:/GopherAI-v2/service/session/session.go#L17-L33)

当前实现说明：

- 返回的会话列表来自后端内存的 AIHelperManager（不是直接查 `session` 表）。
- 因此：后端重启后需要依赖“从 DB 回放消息”才能恢复会话列表。

### 5.2 点击“新聊天”：创建新的 SessionID

前端点击“新聊天”时，先进入临时态：

- `currentSessionId = 'temp'`
- `tempSession = true`

见 [createNewSession](file:///d:/GopherAI-v2/vue-frontend/src/views/AIChat.vue#L201-L209)

当用户第一次在新聊天里发送问题时：

- 流式模式调用：`POST /api/v1/AI/chat/send-stream-new-session`
- 非流式模式调用：`POST /api/v1/AI/chat/send-new-session`

后端会创建 session（uuid）并写入 DB，随后把 `sessionId` 返回给前端：

- 流式：先通过 SSE 输出 `data: {"sessionId":"..."}\n\n`（见 [CreateStreamSessionAndSendMessage](file:///d:/GopherAI-v2/controller/session/session.go#L102-L115)）
- 非流式：在 JSON 响应里直接返回 `sessionId`（见 [CreateSessionAndSendMessage](file:///d:/GopherAI-v2/controller/session/session.go#L65-L85)）

### 5.3 点击“旧会话”：SessionID 如何切换

前端点击旧会话条目会调用：

- `switchSession(sessionId)`

见 [switchSession](file:///d:/GopherAI-v2/vue-frontend/src/views/AIChat.vue#L211-L236)

它会：

1. `currentSessionId = sessionId`（切换当前会话）
2. 若该会话消息未缓存，则调用 `POST /api/v1/AI/chat/history` 拉历史
3. `currentMessages = sessions[sessionId].messages`，刷新右侧聊天窗口

### 5.4 获取历史：/AI/chat/history

前端在切换会话时或手动同步历史时会调用：

- `POST /api/v1/AI/chat/history`
- body：`{ "sessionId": "<sid>" }`

后端 controller： [ChatHistory](file:///d:/GopherAI-v2/controller/session/session.go#L166-L183)

后端 service： [GetChatHistory](file:///d:/GopherAI-v2/service/session/session.go#L170-L191)

实现说明：

- 当前历史数据来自内存 `AIHelper.messages`，并非直接查 DB。
- 如果某个会话在当前后端进程内存里不存在，会返回失败。

## 6. 聊天发送与流式输出（最核心链路）

### 6.1 非流式发送

前端调用：

- `POST /api/v1/AI/chat/send`
- body：`{ "sessionId": "<sid>", "question": "...", "modelType": "1" }`

后端链路：

1. controller：[ChatSend](file:///d:/GopherAI-v2/controller/session/session.go#L121-L140)
2. service：[ChatSend](file:///d:/GopherAI-v2/service/session/session.go#L148-L168)
3. 通过 `GetOrCreateAIHelper(userName, sessionId, modelType, config)` 获取会话专属 AIHelper（见 [AIHelperManager](file:///d:/GopherAI-v2/common/aihelper/manager.go#L23-L50)）
4. AIHelper 生成回复：
  - 先 `AddMessage(userQuestion, ..., Save=true)` 存用户问题并通过 MQ 异步入库
  - 把消息历史转换为 `[]*schema.Message`（role+content）：[ConvertToSchemaMessages](file:///d:/GopherAI-v2/utils/utils.go#L52-L66)
  - 调模型 `GenerateResponse`
  - 再 `AddMessage(aiAnswer, ..., Save=true)` 存 AI 回复并入库
  - 见 [GenerateResponse](file:///d:/GopherAI-v2/common/aihelper/aihelper.go#L66-L89)

### 6.2 流式发送（SSE）

前端调用：

- `POST /api/v1/AI/chat/send-stream`
- body：`{ "sessionId": "<sid>", "question": "...", "modelType": "1" }`

后端链路：

1. controller： [ChatStreamSend](file:///d:/GopherAI-v2/controller/session/session.go#L142-L165)
2. service： [StreamMessageToExistingSession](file:///d:/GopherAI-v2/service/session/session.go#L84-L130)
3. service 构造 `cb := func(msg string){ writer.Write("data: "+msg+"\n\n"); Flush() }`
4. AIHelper 调模型的 `StreamResponse`，模型每收到一个 chunk 就回调 `cb(chunk)`：
  - 见 OpenAI 实现 [StreamResponse](file:///d:/GopherAI-v2/common/aihelper/model.go#L61-L86)
5. 流结束后，后端再发一条结束标记：
  - `data: [DONE]\n\n`（见 [StreamMessageToExistingSession](file:///d:/GopherAI-v2/service/session/session.go#L122-L128)）

前端判断流结束：

- 收到 `data: [DONE]` 就结束本次流（见 [AIChat.vue](file:///d:/GopherAI-v2/vue-frontend/src/views/AIChat.vue#L381-L387)）

## 7. 模型类型 modelType（1/2/3/4）

modelType 是字符串编号，由前端选择后随请求一起发给后端。

模型工厂映射关系： [factory.go](file:///d:/GopherAI-v2/common/aihelper/factory.go#L33-L66)

- `"1"`：OpenAI（OpenAI 兼容接口）
- `"2"`：阿里百炼 RAG（通过 Redis 向量检索拼 Prompt）
- `"3"`：MCP（先让 LLM 产出工具调用 JSON，再调用 MCP 服务，再二次生成）
- `"4"`：Ollama（本地模型接口）

OpenAI 兼容接口所需环境变量：

- `OPENAI_API_KEY`
- `OPENAI_MODEL_NAME`
- `OPENAI_BASE_URL`

见 [NewOpenAIModel](file:///d:/GopherAI-v2/common/aihelper/model.go#L37-L51)

## 8. 文件上传与 RAG（简述）

当前实现依赖 ONNXRuntime，并且模型/label 路径在服务层写死为 Linux 路径（见 [service/image.go](file:///d:/GopherAI-v2/service/image/image.go#L11-L23)），容器里需要准备对应文件路径才能使用。

## 10. 最小可用的用户操作顺序（建议）

1. 注册（发验证码 → 注册 → 拿 token & username）
2. 登录（得到 token，前端保存 localStorage）
3. 进入聊天页（加载会话列表）
4. 点“新聊天”（进入临时态）
5. 发送第一条消息（后端创建 sessionId 并返回）
6. 继续聊天（每次请求携带 sessionId + token）
7. 点击旧会话（切换 currentSessionId，必要时拉历史）
